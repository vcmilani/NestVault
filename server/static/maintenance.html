Limpeza Noturna
POST /maintenance/nightly-cleanup
Médio risco
Aplica a política de retenção progressiva em todas as versões: últimas 24h tudo é preservado; entre 1 dia e 1 mês, 1 versão done por dia; entre 1 e 6 meses, 1 por semana; acima de 6 meses, 1 por mês. Versões failed/incomplete são removidas imediatamente se houver versão done mais recente. Executa automaticamente à meia-noite — use este botão para acionar manualmente.
⊘
Limpeza de Órfãos
POST /maintenance/cleanup-orphans
Baixo risco
Remove todos os FileContents não referenciados por nenhuma versão ativa. Libera espaço em disco ocupado por conteúdo sem referência. Executa em background — acompanhe o progresso na página de Atividades.
⇄
Re-replicar
POST /maintenance/rereplicate
Baixo risco
Cria cópias faltantes para conteúdos com menos réplicas que o REPLICATION_FACTOR. Útil após adicionar um novo volume de storage.
⇌
Reconciliar Replicação
POST /maintenance/reconcile-replication
Médio risco
Operação em duas fases: remove cópias excedentes (acima do fator) e preenche conteúdos sub-replicados. Garante conformidade exata com REPLICATION_FACTOR.
⚿
Cifrar Existentes
POST /maintenance/encrypt-existing
Irreversível
Cifra em AES-256-GCM todos os FileContents ainda não cifrados. Requer ENCRYPTION_ENABLED=true no servidor. Operação irreversível — não é possível desfazê-la.
⊟

Migrar Disco
POST /maintenance/migrate-disk
Alto risco
Move todos os dados de um disco de origem para um ou mais discos de destino. Valida espaço disponível e integridade SHA-256 antes de remover a cópia original.
✓
Validação de Integridade
POST /maintenance/validate-integrity
Médio risco
Verifica se todos os arquivos das últimas versões done existem fisicamente no disco. Versões com arquivos ausentes são marcadas como failed e seus registros removidos do banco — permitindo que o backup reenvie os arquivos na próxima execução.
⛁
Backup do Banco de Dados
POST /maintenance/db-backup
Baixo risco
Exporta o banco de dados (PostgreSQL ou SQLite) para o diretório _db_backups/ em cada volume de storage saudável. Mantém os últimos 7 backups por volume (configurável via DB_BACKUP_RETENTION). Executa automaticamente todo dia às 01:00 — use este botão para acionar manualmente. Sem o backup, uma falha no SSD tornaria os arquivos dos HDDs irrecuperáveis.
